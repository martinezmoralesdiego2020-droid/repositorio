# Git Ejemplo

Repositorio de ejemplo para practicar Git, ramas, merges y commits.

## Archivos del proyecto

- **codigo1.py**: Primer archivo del proyecto, creado en la rama principal.
- **codigo2.py**: Segundo archivo añadido al proyecto y compartido entre las ramas.
- **codigo3.py**: Archivo agregado inicialmente en `rama1` y posteriormente fusionado con la rama `main`.

## Historial de commits

| Commit | Mensaje | Descripción |
| ------ | ------- | ----------- |
| `6fceaa1` | cambio para prueba ssh | Actualización del repositorio utilizando autenticación SSH. |
| `8573cb0` | Cambia el archivo.py | Modificación de archivos dentro de la rama principal. |
| `8cdd236` | cambios en main | Actualización general realizada en la rama `main`. |
| `ea428ec` | Merge branch | Fusión de `rama1` con la rama principal `main`. |

## Ramas

- **main**: Rama principal del proyecto. Contiene `codigo1.py`, `codigo2.py` y `codigo3.py`.
- **rama1**: Rama donde se trabajó inicialmente con `codigo3.py` antes del merge.
- **rama2**: Rama secundaria que contiene `codigo1.py` y `codigo2.py`.

## Cómo usar

Este repositorio es únicamente para práctica de Git y Markdown. Se pueden crear ramas, realizar cambios, ejecutar merges y practicar resolución de conflictos.
